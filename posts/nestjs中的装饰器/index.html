1 Provider、Controller和Module

Provider、Contoller和Module三个装饰器是NestJS中最基础的三个装饰器，分别对应模块、控制器和服务三个概念

模块Module： Nest程序由一个个模块组成， 模块中包括控制器和服务

控制器 Controller: 用来处理路由， 控制器其实就是一个类， 其中的每个方法称为Handler, 用来调用处理对应的业务逻辑， 通常将业务逻辑封装在Server中处理

服务Servicer: 处理业务逻辑

Nest通过Module装饰器来创建模块，通过 @Controller、@Injectable 分别声明其中的 controller 和 provider

import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
    imports: [],
    controllers: [AppController],
    providers: [
        AppService,
        {
            provide: 'APP_CONFIG',
            useValue: 'app config value'
        }
    ],
})
export class AppModule { }

import { Controller, Get, Inject } from '@nestjs/common';
import { AppService } from './app.service';

@Controller()
export class AppController {
    // 通过属性的方式注入
    @Inject(AppService)
    private readonly appService: AppService

    @Get()
    getHello(): string {
        return this.appService.getHello();
    }
}

/// 其中注入的方式可以是构造器注入， 也可以是属性注入
@Controller()
export class AppController {
    // 通过属性的方式注入
    @Inject(AppService)
    private readonly appService: AppService

    @Get()
    getHello(): string {
        return this.appService.getHello();
    }
}

// 通过构造器来注入
@Controller()
export class AppController {
    constructor(private readonly appService: AppService) { }

    @Get()
    getHello(): string {
        return this.appService.getHello();
    }
}

属性注入时需要写明对应的token, 注入其中的可以是class也可以是其他类型, 当token不存在时会报错， 可以使用Optional装饰器来声明为可选属性

@Module({
    imports: [],
    controllers: [AppController],
    providers: [
        AppService,
        {
            provide: 'APP_CONFIG',
            useValue: 'app config value'
        },
        {
            provide: "SOME_OTHER_SERVICE",
            useFactory() {
                return {
                    name: 'Alanfliu'
                }
            }
        }
    ],
})

// app.serice.ts
import { Inject, Injectable, Optional } from '@nestjs/common';

@Injectable()
export class AppService {
    @Inject('APP_CONFIG')
    private readonly appConfig: string

    @Optional()
    @Inject('SOME_OBJECT_SERVICE')
    private readonly someService: Record<string, any>

    getHello(): string {
        return 'Hello World!' + this.appConfig + this?.someService?.name;
    }
}

将服务提供给其他模块使用时， 需要手动声明exports

@Module({
    imports: [],
    controllers: [AppController],
    providers: [
        AppService,
        {
            provide: 'APP_CONFIG',
            useValue: 'app config value'
        },
        {
            provide: "SOME_OTHER_SERVICE",
            useFactory() {
                return {
                    name: 'Alanfliu'
                }
            }
        }
    ],
    exports: [AppService] // 此时其他模块可以使用该服务
})

如果模块被很多地方都引用，为了方便，可以用 @Global 把它声明为全局的，这样它 exports 的 provider 就可以直接注入了

@Global()
@Module({
    imports: [],
    controllers: [AppController],
    providers: [
        AppService,
        {
            provide: 'APP_CONFIG',
            useValue: 'app config value'
        },
        {
            provide: "SOME_OTHER_SERVICE",
            useFactory() {
                return {
                    name: 'Alanfliu'
                }
            }
        }
    ],
    exports: [AppService]
})

2 路由参数处理

处理路由， 首先需要对应的请求这么处理以及怎么获取客户端传来的数据

  1. 通过@Query 参数来获取URL后的路径参数， 比如http://localhost:3000/bbb?name=alanfliu&gender=男
@Controller()
export class AppController {
    constructor(private readonly appService: AppService) { }

    @Get()
    getHello(): string {
        return this.appService.getHello();
    }

    @Get('bbb')
    getBBBURLParams(@Query() params: Record<string, any>) {
        return params;
    }
}

此时， Query装饰器会将路径参数解析成一个对象， 比如上述代码的例子， 会将params参数解析成下面的结果：

{"name":"alanfliu","gender":"男"}
  1. 通过Post装饰器来解析Post请求， 通过Body装饰器来获取参数
@Controller()
export class AppController {
    constructor(private readonly appService: AppService) { }

    @Get()
    getHello(): string {
        return this.appService.getHello();
    }

    @Get('bbb')
    getBBBURLParams(@Query() params: Record<string, any>) {
        return params;
    }

    @Post('aaa')
    getAAAPostData(@Body() params: Record<string, any>) {
        return {
            flag: "这是服务端数据",
            ...params
        };
    }
}

当发送JSON数据为: {“name”: “alanfliu”, “school”: “CQUPT”}时， 会返回如下结果

{
	"flag": "这是服务端数据",
	"name": "JSON",
	"school": "CQUPT"
}

除了 @Get、@Post 外，还可以用 @Put、@Delete、@Patch、@Options、@Head 装饰器分别接受 put、delete、patch、options、head 请求

4 总结

  • @Module： 声明 Nest 模块
  • @Controller：声明模块里的 controller
  • @Injectable：声明模块里可以注入的 provider
  • @Inject：通过 token 手动指定注入的 provider，token 可以是 class 或者 string
  • @Optional：声明注入的 provider 是可选的，可以为空
  • @Global：声明全局模块
  • @Param：取出 url 中的参数，比如 /aaa/:id 中的 id
  • @Query: 取出 query 部分的参数，比如 /aaa?name=xx 中的 name
  • @Body：取出请求 body，通过 dto class 来接收
