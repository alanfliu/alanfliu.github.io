前言

在写一段时间Node程序后， 我发现最难的一部分还是后端生态， 这部分才是主要需要攻克的地方。因此想借助学习NestJS框架，顺便把后端相关的生态整理一遍， 包括Docker、Redis、Mysql、ORM、RabbitMQ等等。

博客里NestJS相关的知识以一个学生管理系统为例， 来系统的记录该系统从一个Demo慢慢的壮大、健全的过程

光说不练假把式， 我还是觉得光是看一些教程， 没有一个需求来承载知识， 这个知识很会忘记， 学生管理系统是我学生时代写的第一个稍微大一点的程序， 还记得当初用C语言写完之后， 非常自豪的跟室友炫耀。 现在想把这个程序完善一下， 成为一个前后端健全的Web应用。

1 需求分析与技术选型

1.1 需求分析

该系统该开始不需要构建非常完善， 把应用跑起来再说，其基本的需求如下：

  1. 分为两个角色：学生、老师
  2. 学生可以在系统上选课， 选完课进入班级， 在班级界面可以查看学习进程，以及最终的考试成绩
  3. 老师可以系统上发布课程，等待学生来选课，然后组件班级； 老师可以管理课程进度， 发送一些资料； 在考试后可以发布学生成绩
  4. 学生和老师可以互相发送信息， 来讨论课程相关的知识点。

1.2 技术选型

  • 服务开发：NestJS + Typescript
  • 存储： Mysql + Prisma + Redis
  • 消息队列: RabbitMQ
  • 前端： React + Typescript + TailwindCss + React Redux + React Router
  • 部署： Docker + PM2
